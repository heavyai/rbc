JITRemoteCodeLibrary¶

class rbc.irtools.JITRemoteCodeLibrary(codegen: numba.core.codegen.BaseCPUCodegen, name: str)¶

JITRemoteCodeLibrary was introduce to prevent numba from calling functions that checks if the module is final. See xnd-project/rbc issue #87.

get_pointer_to_function(name)¶

We can return any random number here! This is just to prevent numba from trying to check if the symbol given by “name” is defined in the module. In cases were RBC is calling an external function (i.e. allocate_varlen_buffer) the symbol will not be defined in the module, resulting in an error.
