get_client_config¶

rbc.omniscidb.get_client_config(**config)¶

Retrieve the omnisci client configuration parameters from a client home directory.

Note that here the client configurations parameters are those that are used to configure the client software such as rbc or pymapd. This is different from omnisci instance configuration described in https://docs.omnisci.com/latest/4_configuration.html that is used for configuring the omniscidb server software.

In Linux clients, the omnisci client configuration is read from: $HOME/.config/omnisci/client.conf

In Windows clients, the configuration is read from %UserProfile/.config/omnisci/client.conf or %AllUsersProfile/.config/omnisci/client.conf

When OMNISCI_CLIENT_CONF environment variable is defined then the configuration is read from the file specified in this variable.

The configuration file must use configuration language similar to one used in MS Windows INI files. For omnisci client configuration, the file may contain, for instance:

[user]
name: <OmniSciDB user name, defaults to admin>
password: <OmniSciDB user password>

[server]
host: <OmniSciDB server host name or IP, defaults to localhost>
port: <OmniSciDB server port, defaults to 6274>
Parameters

config (dict) – Specify configuration parameters that override the parameters from configuration file.

Returns

config – A dictionary of user, password, host, port, dbname and other RemoteJIT options.

Return type

dict
